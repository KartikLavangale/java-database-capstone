# User Stories

## 1. Patient User Story
* **Story:** As a Patient, I want to search for doctors by specialty so that I can find the right specialist for my needs.
* **Priority:** High
* **Story Points:** 5
* **Acceptance Criteria:**
  * User can select a specialty from a dropdown.
  * System displays a list of doctors matching the specialty.
  * Each result shows Doctor Name, Specialty, and Availability.

## 2. Doctor User Story
* **Story:** As a Doctor, I want to view my daily appointments so that I can manage my schedule efficiently.
* **Priority:** Medium
* **Story Points:** 3
* **Acceptance Criteria:**
  * Doctor can log in and see a dashboard.
  * Dashboard lists all appointments for the current date.
  * List includes Patient Name, Time, and Status.

## 3. Admin User Story
* **Story:** As an Admin, I want to add new doctors to the system so that patients have more options.
* **Priority:** High
* **Story Points:** 8
* **Acceptance Criteria:**
  * Admin form includes fields for Name, Specialty, Email, and Availability.
  * Submitting the form saves the doctor to the database.
  * System validates that the email is unique.
